Temporal_Fixer — Reconstruction Prompt Gallery

Every reconstruction prompt we tried, with its two best reconstruction examples (left = clean source, right = the frozen editor's per-frame reconstruction). Editor = Qwen-Image-Edit-2509 int4 + Lightning-v2 (nunchaku), 4-step, full denoising from Gaussian noise per frame — never tuned. The fixer (a separate causal video model) is trained only to undo the temporal inconsistency you see on the right; at test the only thing that changes is the editor's prompt (reconstruction → edit). 256 reconstruction clips across 64 prompt variants.

Metrics per variant: Δstr = injected frame-to-frame structural inconsistency; driving Δstr = same on driving footage; frame-corr = across-frame consistency of the injected change (lower = more per-frame re-interpretation, the behavior we want).

Editing for free — the v3 fixer at inference (source | edit-draft | fixed)

Same fixer, trained only on reconstruction. At test the editor's prompt becomes an actual edit (snow / sunset / sunny / night→day); the fixer removes the temporal inconsistency and the edit is inherited. Left→right in each clip: clean source, the per-frame int4 edit draft, the fixer output.
